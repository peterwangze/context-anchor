# 记忆热度机制详解

## 热度计算公式

```
热度 = 初始热度 + 访问加成 - 时间衰减 + 特殊加成
```

## 初始热度

| 来源 | 初始热度 | 说明 |
|------|----------|------|
| 新信息 | 50 | 默认中等热度 |
| 用户标记重要 | 80 | 强制高热度 |
| 从长期记忆加载 | 30 | 冷记忆加载 |
| 子 Agent 返回 | 40 | 派生信息 |
| 错误/教训 | 60 | 重要性较高 |

## 热度变化规则

### 访问加成

| 行为 | 热度变化 | 说明 |
|------|----------|------|
| 在对话中引用 | +10 | 每次引用 |
| 用户确认重要 | +30 | 手动标记 |
| 完成相关任务 | +5 | 任务完成 |
| 跨 session 存活 | +20 | 证明价值 |

### 时间衰减

| 时间 | 热度变化 | 说明 |
|------|----------|------|
| 每小时 | -1 | 持续衰减 |
| 每天未访问 | -5 | 额外衰减 |
| 超过 7 天未访问 | -10 | 快速衰减 |

### 热度上限和下限

- 最大热度：100
- 最小热度：0
- 热度 < 10 且无引用 → 标记可清理

## 热度分级与行为

| 热度范围 | 层级 | 存储位置 | 行为 |
|----------|------|----------|------|
| 80 - 100 | 工作记忆（热） | 内存 | 保持在上下文，定期提醒 |
| 30 - 80 | 短期记忆（温） | session 文件 | 可恢复，按需加载 |
| 0 - 30 | 长期记忆（冷） | 项目文件 | 按需检索，归档保存 |

## 记忆流动示例

### 晋升流程

```
1. 短期记忆（热度 50）
2. 被频繁访问（+10 × 3 = +30）
3. 热度达到 80
4. 晋升到工作记忆
5. 在上下文中保持活跃
```

### 降级流程

```
1. 工作记忆（热度 85）
2. 3 小时未访问（-1 × 3 = -3）
3. 热度降至 82
4. 继续衰减至 80 以下
5. 降级到短期记忆
6. 继续衰减至 30 以下
7. 降级到长期记忆
```

## 热度评估时机

| 时机 | 频率 | 说明 |
|------|------|------|
| heartbeat | 每 30 分钟 | 定期评估 |
| 对话后 | 每 N 次对话 | 增量更新 |
| 任务完成时 | 按需 | 批量更新 |
| session_end | 一次 | 最终评估 |

## 特殊处理

### 强制保留

用户明确标记"记住这个"的信息：
- 热度设为 80（工作记忆下限）
- 跳过自动降级
- 只能手动删除

### 快速遗忘

用户明确标记"忘了这个"的信息：
- 热度设为 0
- 标记为可清理
- 下次清理时删除

### 错误信息

记录为错误/教训的信息：
- 初始热度 60
- 不参与常规衰减
- 晋升到长期记忆时保留完整上下文
